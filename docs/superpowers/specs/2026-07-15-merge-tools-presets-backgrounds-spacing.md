## Feature 1 — Line spacing

There is no multi-line text: "lines" are stacked `TextElement`s at fixed baselines. So the knob
is a stack-gap adjuster, precedented by headshot `lineGap`:

- `DesignOverrides.lineGap?: number` (pt, additive, clamp [-20, 60] in a new shared sanitizer —
  today's `isDesignOverrides` only checks `v===1`; add clamping like `headshot-style-sanitize.ts`).
- In `applyDesign`: order the spec's text elements top→bottom (descending y — PDF y-up), keep the
  first fixed, shift each subsequent one down by `cumulative index * lineGap` (negative pulls
  tighter). Elements at (near-)equal y are one visual line: same shift, no gap between them.
  All four specs are single-column stacks, so this is safe; write the tie tolerance as a
  constant.
- UI: "Line spacing" slider in DesignPanel's Text section (global, not per-slot), default 0.
- Custom-canvas mode is unaffected (boxes are user-positioned).

## Feature 2 — Bundled backgrounds (certificate + ticket)

Backgrounds ship as small vector PDFs (crisp at any size, few KB; `merge-render.ts` already
embeds pdf backgrounds), generated by a checked-in script — not hand-drawn binaries:

- `scripts/gen-backgrounds.mjs` (pdf-lib): 4 designs per tool into
  `public/backgrounds/{certificate,ticket}/<id>.pdf`, committed. Anti-vibecode: neutral
  ivory/grey fields, ONE muted accent (deep navy or warm gold), no gradients/spotlights.
  Certificate: classic double-rule border; single bottom accent bar; corner rules; plain
  centered head rule. Ticket: left stub band with perforation dashes; top accent band; thin
  full border; minimal corner ticks. Page sizes must match each tool's default spec size (read
  `certificateSpec`/`ticketSpec`).
- Registry `lib/design-backgrounds.ts` mirroring `designer-fonts.ts`: `TOOL_BACKGROUNDS`
  (`{id, label, file, tool}`), `loadBackgroundById` → fetch → base64 (pdf convention from
  `assetSrc`), memo cache.
- `DesignOverrides.background?: { id: string } | null` + `applyDesign` (or the client just
  before render) sets `DocumentSpec.background`. Implementer picks the seam that keeps
  `applyDesign` pure (passing loaded bytes in is fine); preview and final render must both
  honor it.
- UI: "Background" section in DesignPanel — "None" + thumbnails (raster the pdf via the
  MergePreview pdfjs path at small size, or static preview PNGs emitted by the same script —
  implementer's choice, cheapest wins). Only rendered for tools with registry entries.
  User-uploaded custom backgrounds stay the F3 custom-canvas feature; this is separate and
  simpler.

## Feature 3 — Design presets / templates

Clone the headshot Look Presets pattern (`lib/headshot-presets.ts` + `app/studio/PresetBar.tsx`):

- `lib/design-presets.ts`: localStorage `ee.designPresets.<toolId>`, DOM-free CRUD
  (`list/create/update/rename/deletePreset`), newest-first, `crypto.randomUUID`.
- Preset payload captures the CURRENT mode: built-in → `{ kind:"design", layoutId, overrides }`;
  custom canvas → `{ kind:"custom", customDesign }` (assetIds keep pointing at the shared
  `ee-design-assets` IndexedDB — assets are never GC'd today, so preset refs stay valid; if an
  asset is missing at apply time, `customDesignToSpec` already drops it gracefully). Plus
  `{ id, name, preview, updatedAt }`.
- Thumbnail: render row-0 via `renderOne` → pdfjs → ~256px PNG data URL (reuse MergePreview's
  raster path as a helper).
- `components/DesignPresetBar.tsx`: same UX as PresetBar (save current look, apply, update,
  rename, delete, active highlight). Apply = one `setOverrides`/`setCustomDesign` call — no
  decomposition needed. Mounted in MergeToolClient AND CertificateClient, above DesignPanel.
- Sample-data caveat: thumbnails render with the tool's `EMPTY_ROW`/sample copy, fine.

## Risks

- applyDesign stack-shift vs per-tool element order: derive order from y, don't assume array
  order; tie tolerance for same-line elements.
- Preset payloads with uploaded session fonts (`upload:` font ids are session-only, never
  persisted): saving a preset that references one must either warn or store the font bytes —
  v1: presets keep the fontId; if missing at apply time the renderer already falls back, add a
  quiet note in the panel ("uploaded fonts are not saved with presets").
- Bundled PDF backgrounds must match spec page sizes or applyDesign pageSize changes — the
  renderer stretches background to the page (`drawBackground` full-cell), so mismatch degrades
  gracefully; generate at default sizes.
